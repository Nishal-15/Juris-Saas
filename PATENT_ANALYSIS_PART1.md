## 1. PRODUCT OVERVIEW

**Product Name:** JurisBot AI Legal Intelligence Platform

**Core Purpose:** A multi-portal, AI-augmented legal SaaS platform that connects Indian citizens with verified lawyers, automates legal case filing, delivers real-time legal AI consultations, and manages the full lifecycle of a legal case from filing to verdict.

**Target Users:**
- Citizens (individuals with legal problems)
- Verified Lawyers / Legal Advocates
- Platform Administrators
- Flutter mobile app users (cross-platform)

**Problems Solved:**
1. Citizens lack access to affordable legal advice
2. No structured way to file and track legal cases digitally
3. Lawyers have no SaaS tool to manage caseloads and clients
4. Legal AI chatbots give generic, inaccurate answers
5. No real-time lawyer-citizen communication infrastructure
6. Legal system is inaccessible to non-English speakers
7. No automated hearing reminders for court dates
8. Lawyer credential verification is manual and slow

**Unique Value Proposition:**
- Only platform combining AI legal Q&A + case filing + lawyer marketplace + real-time video/chat + automated court-date WhatsApp alerts in one system
- Supports 22 scheduled languages of India for voice-based case filing
- AI auto-generates formal legal case titles from plain-language descriptions
- Subscription-gated lawyer access with quota enforcement

## 3. USER FLOWS

### 3A. Citizen Filing a Case
1. Register/Login → JWT stored in localStorage
2. Navigate to "File a Case" → FilingConsole loads
3. Step 1: Select category (Home & Property, etc.)
4. Step 2: Type incident narration — after 2.5s pause OR blur → AI auto-fills title, category, legalType
5. Select incident date via MaterialDatePicker
6. Step 3: Upload evidence (optional)
7. Step 4: Review all fields
8. Click "FINALIZE & FILE CASE" → POST `/cases`
9. Server saves case, runs lawyer matching by specialization
10. Success modal shows matched lawyers
11. Citizen clicks "Connect" → POST `/cases/connect/:caseId/:lawyerId`
12. Case status → "Pending Expert Acceptance"
13. Lawyer gets real-time socket notification + DB notification

### 3B. Lawyer Accepting a Case
1. Login → LawyerDashboard loads all data in parallel
2. Pending Queue shows case request with citizen name, date
3. Lawyer clicks "Accept" → POST `/cases/accept/:caseId`
4. trackingHistory gets "Lawyer Accepted" entry
5. Status → "In Progress"
6. Citizen gets real-time socket notification + WhatsApp AI message
7. Case moves to Active Workspace on dashboard

### 3C. Lawyer Updating a Case
1. Open case from Active Workspace → CaseDetails
2. Update status, hearing date, court location, next steps
3. Add update note → appended to trackingHistory with timestamp
4. PATCH `/cases/:id/management`
5. Citizen gets real-time push + WhatsApp alert
6. Citizen's timeline view shows new milestone with exact timestamp

### 3D. Hearing Reminder (Automated)
1. Cron job runs at 8:00 AM daily
2. Queries cases where hearingDate = today or +48h
3. Generates personalized AI reminder text per case
4. Sends WhatsApp to both citizen and lawyer phone numbers

### 3E. Admin Verifying a Lawyer
1. Admin logs in with admin_master_token or admin JWT
2. Verification Queue shows pending lawyers with credentials
3. Admin clicks Approve → PATCH `/admin/verify-lawyer/:id`
4. `isVerified: true`, `verificationStatus: "verified"`
5. Lawyer now appears in citizen marketplace

## 5. CORE ALGORITHMS

### 5A. AI Case Triage (analyze-story)
- Input: free-text description (min 10 chars)
- Sends to Groq llama-3.3-70b with a legal taxonomy of 60+ case types
  - Civil: property dispute, breach of contract, money recovery, tenant dispute, etc.
  - Criminal: theft, cheating, domestic violence, cybercrime, etc.
  - Corporate: shareholder dispute, insolvency, merger dispute, etc.
- Output: `{ title (formal court language), category, legalType }`
- Fallback 1: Gemini API
- Fallback 2: Heuristic — first 7 words + "..."

### 5B. Lawyer-Specialization Matching
- On case creation: regex-match legalType against Lawyer.specialization
- Returns top 3 matches sorted by rating
- Shown immediately in success modal

### 5C. Open Case Marketplace Filtering
- Splits lawyer specialization by "&" or ","
- Runs `$or` query on type OR title (not description — prevents bleed)
- Fallback: if fewer than 5 results → returns ALL unassigned cases

### 5D. Subscription Quota Enforcement
- `checkSubscription` middleware intercepts case-accept requests
- Counts cases where `assignedLawyer = lawyerId AND createdAt >= subscriptionStartedAt`
- Syncs count to lawyer document
- Blocks with 403 if `count >= caseLimit AND caseLimit != 9999`

### 5E. Chronological Timeline Sort
- Merges static "Case Filed & Registered" (createdAt) with all trackingHistory entries
- Sorts ascending: `new Date(a.date) - new Date(b.date)`
- Last item → `active` class (gold pulsing dot)

### 5F. Hearing Countdown Algorithm
- Strips time from both current date and hearingDate (midnight normalization)
- Difference in ms → calendar days
- `days < 0` → "Hearing date has passed"
- `days === 0` → "Your hearing is TODAY"
- `days <= 3` → red urgent styling
- `days > 3` → gold normal styling

## 7. SECURITY MECHANISMS

| Mechanism | Detail |
|-----------|--------|
| JWT Auth | 1-day expiry (users), 7-day (lawyers); dual-header: `Authorization: Bearer` OR `x-auth-token` |
| Admin bypass | Static `admin_master_token` for institutional access |
| RBAC | `auth(["lawyer"])`, `auth(["admin"])`, `auth()` (any authenticated) |
| Case scoping | Owner OR assigned lawyer OR admin OR open marketplace case |
| Password | bcrypt, 10 salt rounds; stripped from all responses |
| Min password | 6 characters; email regex validation |
| Cross-collection dupe check | Email checked in both User AND Lawyer collections |
| File sanitization | Spaces→underscores, special chars stripped; separate upload dirs per type |
| CORS | Dynamic — accepts any origin (suitable for Vercel + custom domains) |

## 8. EDGE CASES & FAIL-SAFES

| Scenario | Handling |
|----------|----------|
| Groq API fails | Falls back to Ollama if `USE_OLLAMA=true`, else returns debug error message |
| AI triage fails | Heuristic fallback: first 7 words of description |
| Lawyer matching returns 0 | "No immediate matches" shown; case is still filed successfully |
| WhatsApp API fails | Simulation mode: logs to console only |
| OTP expired | "OTP expired or invalid" returned |
| Case already assigned | 400 "Already assigned" |
| Subscription limit hit | 403 from checkSubscription middleware |
| Socket disconnect | "User disconnected from workspace" logged |
| Video call rejected | `end-call` event stops peer connection + all media tracks |
| Case not found | 404 with descriptive message |
| Unauthorized case access | 403 "You are not authorized to view this legal file" |
| Missing Agora certificate | 500 with error detail |
| Upload dir missing | Auto-created with `fs.mkdirSync({ recursive: true })` |

## 9. DIFFERENTIATING FACTORS (PATENT-RELEVANT)

1. **AI-Triggered Legal Auto-Fill:** Debounced real-time analysis of plain-language descriptions → automatic generation of formal court-language case titles using a legal taxonomy of 60+ case types

2. **Two-Phase Lawyer-Citizen Acceptance Protocol:** Citizen requests specific lawyer → "Pending Expert Acceptance" status → lawyer accepts → "In Progress" — auditable chain tracked in timeline

3. **AI-Generated WhatsApp Hearing Reminders:** Cron generates unique AI-written reminders per case, delivered to both parties via WhatsApp — not generic SMS blasts

4. **Calendar-Day Hearing Countdown:** Midnight normalization prevents timezone-based false "TODAY" alerts

5. **Live Subscription Quota Sync:** Lawyer case quota recomputed from DB on every API call — no caching, prevents over-claiming

6. **22-Language Voice Case Filing:** Web Speech API with all 22 Indian scheduled languages

7. **Specialization-Filtered Marketplace with Smart Fallback:** Keyword matching on type/title only (not description), auto-expands if fewer than 5 results

8. **Multi-Engine AI Cascade:** Groq → Ollama → heuristic — 100% uptime for AI features

9. **Zero-Jump Sidebar:** `transform + margin-left` sync with `--dur: 0s` pre-paint state restoration prevents FOUC

## 10. CLAIMABLE INVENTIONS

### System Claims
1. A multi-portal legal case management system comprising a citizen portal, lawyer portal, admin portal, and AI service communicating via REST API and WebSocket, wherein the system maintains a chronological audit trail of all case status changes with timestamps

2. A system for automated lawyer-citizen matching wherein, upon case submission, the system queries a lawyer database using specialization regex matching against the case's legal classification and returns a ranked list of matching advocates

3. A subscription enforcement system for a legal marketplace wherein a middleware component intercepts lawyer case-acceptance requests, dynamically computes the lawyer's case count within the current billing cycle, and denies access when the quota is reached

### Method Claims
1. A method for automated legal case title generation: receiving plain-language incident description → debounce timer detects input cessation → AI model with domain-specific legal taxonomy generates formal court title → form fields auto-populated

2. A method for AI-generated WhatsApp hearing reminders: scheduled process scans cases with hearingDate within 48h → generates personalized AI text per case → delivers individualized messages to both lawyer and citizen

3. A method for real-time bidirectional legal case tracking: lawyer updates case status → socket event pushed to citizen's room → persistent notification created → AI WhatsApp alert sent → timestamped milestone appended to trackingHistory

### Workflow Claims
1. A lawyer credential verification workflow: lawyer submits registration with bar council certificate → background AI verification initiated → admin review queue → approve/reject → verified lawyers appear in marketplace

2. A two-phase legal consultation engagement workflow: citizen files case and selects lawyer → "Pending Expert Acceptance" status → real-time lawyer notification → lawyer accepts → "In Progress" with trackingHistory entry → both parties notified

## 11. LIMITATIONS & IMPROVEMENTS

### Current Constraints
- OTP stored in-memory (lost on server restart)
- No real payment gateway (UPI deep link is simulated)
- AI service not containerized with main server
- No end-to-end encryption on chat messages
- No OCR or AI analysis of uploaded documents
- Lawyer location hardcoded to "NEW DELHI, INDIA"
- No two-factor authentication
- No case withdrawal mechanism for citizens
- Admin master token is a static string (security risk)
- No rate limiting on API endpoints

- Redis for OTP storage and session management
- Razorpay/Stripe integration for real payments
- Docker Compose for full service orchestration
- Signal Protocol for end-to-end encrypted messaging
- AI-powered document analysis (PDF → case insights)
- GPS-based lawyer location detection
- TOTP-based 2FA for lawyers
- Case withdrawal and dispute resolution workflow
- Multi-language AI responses (Tamil, Hindi, etc. natively)
- Firebase FCM push notifications for mobile app
